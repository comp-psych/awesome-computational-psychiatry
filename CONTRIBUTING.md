# Contributing

This awesome list is community-maintained. Contributions are welcome and encouraged.

## How to Contribute

### Adding a Resource
- Open a pull request adding your resource to the appropriate category
- Use the format: `- [Name](URL) — Brief description`
- Place your entry **alphabetically** within its category
- One resource per pull request makes review easier

### Suggesting a New Category
- Open an issue with the title: `[Category Suggestion] — Name`
- Include: 3–5 candidate resources that would populate the category
- Explain how this category connects to computational psychiatry

### Reporting a Broken Link
- Open an issue with the title: `[Broken Link] — Resource Name`
- Include: the current URL and, if possible, an updated URL

### Suggesting a Removal
- Open an issue with the title: `[Removal] — Resource Name`
- Explain why the resource should be removed (e.g., no longer maintained, content removed, not relevant)

## Guidelines

- **Relevance.** Every resource must be directly relevant to computational psychiatry or closely adjacent fields (computational neuroscience, psychiatric neuroimaging, reinforcement learning for clinical research, etc.)
- **Quality.** Prefer well-established, peer-reviewed, or widely-used resources over niche or unvetted ones
- **Working links.** Verify that your URL resolves before submitting
- **No duplicates.** Check that the resource isn't already listed
- **Brief descriptions.** Keep descriptions to one sentence — informative but concise
- **Academic papers.** Must be peer-reviewed or published as recognized preprints (bioRxiv, PsyArXiv, arXiv)
- **Software.** Must be actively maintained (commits within the last 2 years) or a recognized standard in the field

## Pull Request Checklist

Before submitting your PR, confirm:

- [ ] The resource is relevant to computational psychiatry
- [ ] The URL is working and points to the correct resource
- [ ] The entry follows the format: `- [Name](URL) — Brief description`
- [ ] The entry is placed alphabetically within its category
- [ ] No duplicate entries exist in the list
- [ ] The description is concise (one sentence)

## Code of Conduct

Be respectful, constructive, and collegial. This is an open-source academic resource — we welcome contributions from students, researchers, and clinicians at all levels.
